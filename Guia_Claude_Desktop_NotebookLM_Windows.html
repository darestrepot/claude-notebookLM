Claude Desktop MCP NotebookLM

Conectar Claude Desktop con NotebookLM

Guía paso a paso para Windows. Al terminar, podrás pedirle a Claude que liste tus notebooks, agregue fuentes, consulte tus documentos y genere resúmenes, podcasts o infografías — sin salir del chat.

Antes de empezar
Léelo antes de instalar nada.
  • No es una integración oficial de Google. Funciona sobre APIs internas no documentadas de NotebookLM, que pueden cambiar sin aviso. Estable para uso personal, no para flujos críticos.
  • Autenticas con tu cuenta de Google. El servidor extrae las cookies de tu sesión. Usa una cuenta personal, no la institucional, por si Google modifica algo del lado de NotebookLM.
  • Tiempo estimado: 15 a 20 minutos.
  • Requisitos: Claude Desktop instalado (y abierto al menos una vez) y permiso para ejecutar comandos en PowerShell.
Instalación
1

Instalar uv

uv es el gestor de paquetes de Python con el que corre el servidor. Abre PowerShell y ejecuta:

PS> powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Cierra y vuelve a abrir PowerShell para que uv quede disponible.

2

Instalar el servidor

PS> uv tool install notebooklm-mcp-cli

Esto instala dos herramientas: nlm (línea de comandos) y notebooklm-mcp (el servidor que usará Claude). Verifica con:

PS> uv tool list

Deberías ver notebooklm-mcp-cli v0.7.x con nlm y notebooklm-mcp debajo.

3

Autenticarte con NotebookLM

PS> nlm login

Se abrirá un navegador. Inicia sesión en tu cuenta de Google; las cookies se extraen automáticamente. Confirma con:

PS> nlm login --check
4

Configurar Claude Desktop (manual)

El comando automático no sirve para Claude Desktop. nlm setup add claude-desktop falla porque Claude Desktop no está en la lista de clientes soportados (solo Claude Code). La configuración se hace a mano — es sencilla.

4.1. Averigua la ruta exacta del ejecutable:

PS> where.exe notebooklm-mcp

Copia la ruta que devuelve (algo como C:\Users\TU_USUARIO\.local\bin\notebooklm-mcp.exe).

4.2. Abre el archivo de configuración desde dentro de Claude Desktop: Configuración → Desarrollador → "Editar configuración".

Hazlo siempre por este botón. Si tienes Claude Desktop de la Microsoft Store, el archivo real vive en una ruta virtualizada (…\Packages\Claude_…\LocalCache\Roaming\Claude\). Abrirlo a mano desde el explorador puede llevarte a una copia equivocada que Claude no lee — es el error más común.

4.3. Agrega el bloque mcpServers. El archivo completo debe ser un solo objeto JSON. Si ya tiene una sección preferences, agrega mcpServers como hermano, separado por una coma. El final del archivo debe quedar así:

  },
  "mcpServers": {
    "notebooklm-mcp": {
      "command": "C:\\Users\\TU_USUARIO\\.local\\bin\\notebooklm-mcp.exe"
    }
  }
}

Dos detalles que causan casi todas las fallas:

  • Barras dobles en la ruta (\\), no simples.
  • Una sola llave } de cierre al final. No debe haber un } seguido de un { en medio: eso parte el archivo en dos objetos y Claude lo ignora por completo. En VS Code, no debe quedar nada subrayado en rojo.

Guarda con Ctrl+S.

5

Reiniciar Claude Desktop por completo

Claude Desktop solo lee la configuración al arrancar desde cero. Cerrar la ventana no basta: sigue corriendo en segundo plano.

  • Clic derecho en el ícono de Claude en la bandeja del sistema (junto al reloj; pulsa la flecha ^ si está oculto) → Salir / Quit.
  • Si tras reabrir sigue sin reconocerlo, abre el Administrador de tareas (Ctrl+Shift+Esc), busca procesos "Claude" y termínalos. En algunos equipos hay que reiniciar el computador para que se cierre del todo.
  • Vuelve a abrir Claude Desktop.
6

Verificar

En Configuración → Desarrollador debería aparecer notebooklm-mcp en la lista de servidores MCP locales, con su estado. Luego, en un chat nuevo, escribe:

Lista todos mis notebooks de NotebookLM.

Si te los devuelve, la conexión está completa.

Notas de uso

Cuota. El plan gratuito permite ~50 consultas al día.

Contexto. El servidor expone 35 herramientas, que consumen bastante contexto. Conviene desactivarlo cuando no estés usando NotebookLM.

Mantenimiento. Si vuelves a editar el JSON, no hace falta reiniciar el equipo: basta cerrar Claude desde la bandeja (o terminar el proceso en el Administrador de tareas) y reabrir.

Proyecto de referencia: notebooklm-mcp-cli (jacob-bd) en GitHub. Herramienta comunitaria, sin soporte oficial de Google ni de Anthropic.
